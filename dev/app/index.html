Backend System Design

Backend would need to have restend points for the front-end to get task data. The endpoint will return JSON data.

Rest End Point

TaskList

  • GET /tasklists : returns all task list associated to the user
  • POST /tasklists : create a task list
  • DELETE /tasklists/--tasklist Id-- : deletes the taskList

Tasks

  • GET /tasks/--taskList ID-- : gets all task for task list
  • GET /tasks/--taskList ID--/--task ID-- : gets a task
  • POST /tasks/--taskList ID-- : creates a task in specified taskList
  • DELETE /tasks/--taskList ID--/--task id>-- : deletes a task
  • UPDATE /tasks/--taskList ID--/--task id-- - updates task, marked complete, edit name and description.

Front-end System

Left Menu

The front-end will use the rest endpoints for all data interaction. On page load it should make an ajax call to the rest endpoint to get all task list. On the ajax callback it should create the left menu bar with all the users task list.When the user clicks a specfic task list link in the left menu it will trigger an ajax call to get all task associated to that task list. This will then populate the right box with tasks. The "new Task List" link will display an overlay to input information to create a new task list. The "new task" link will also open an overlay to input information for a new task. There will be a submit button that will trigger an ajax call to either create a new task list or create a new task for a specified task list.

Task Content Area

The complete task link will trigger an ajax call to update the task.
