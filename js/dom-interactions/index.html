JavaScript > DOM interactions

Assume you have jQuery or Prototype, write performance-wise script to:

hide all even rows
$(".even").hide();
do same assuming there are no class names
$("ul > li.nth-child(even)").hide(); // I've created a test file benchmarking a few ways to // hide the event rows. My initial reaction to this was that the fastest way to hide // the even rows was to inject CSS (though that makes things annoying to maintain). // With a huge amount of rows (50,000 in test file), injecting CSS prevents the // javascript from locking up. $("head").append("<style>ul > li:nth-child(even) { display: none; }</style>"); Evan's test file (open console, some tests can take 30 seconds)
Build two arrays containing odd and even rows respectively.
// jquery objects var evens = $(".even"); var odds = $(".odd"); // or dom elements var evens = $(".even").map(function() { return $(this).get(0); }); var odds = $(".odd").map(function() { return $(this).get(0); });
Rearrange nodes in reverse order
$("ul").html($("ul > li").get().reverse());

Ability to demonstrate few possible ways to accomplish this is a plus.
